# KAUST Sparse Spectral Recovery Test

Скрипт для тестирования метода разреженного восстановления спектров по мотивам Arad & Ben-Shahar на гиперспектральных `.mat`-кубах из датасета KAUST.

Сценарий:
- загружает гиперспектральные кубы из папки;
- автоматически находит 3D-массив в каждом `.mat`-файле;
- приводит данные к формату `H × W × B`;
- при необходимости пересэмплирует спектральные каналы до заданного числа полос;
- строит словарь спектров с помощью K-SVD по обучающим кубам;
- моделирует RGB-наблюдения через матрицу спектрального отклика;
- восстанавливает гиперспектральные сигналы из RGB через OMP;
- считает метрики качества и сохраняет итоговый отчёт в JSON.

## Возможности

- Поддержка папки с несколькими `.mat`-файлами.
- Автоматический поиск подходящего 3D-массива внутри `.mat`.
- Обработка входных кубов как в формате `H × W × B`, так и `B × H × W`.
- Пересэмплирование до фиксированного числа спектральных каналов.
- Генерация стандартной RGB response matrix, если своя матрица не передана.
- Обучение словаря спектров через K-SVD.
- Разреженное восстановление через Orthogonal Matching Pursuit.
- Подсчёт `RMSE`, `RRMSE`, `SAM`, `PSNR`.
- Сохранение подробных результатов по сценам и усреднённых метрик.

## Структура входных данных

Скрипт ожидает папку, содержащую `.mat`-файлы:

```text
dataset_root/
├── scene_01.mat
├── scene_02.mat
├── scene_03.mat
└── ...
```

В каждом `.mat`-файле должен находиться хотя бы один трёхмерный массив с гиперспектральным кубом.

Приоритет имён переменных при автопоиске:
- `cube`
- `hsi`
- `HSI`
- `img`
- `data`
- `rad`
- `reflectance`

Если таких имён нет, скрипт выберет подходящий 3D-массив эвристически.

## Быстрый старт

Пример запуска:

```bash
python test_arad_kaust_sparse_recovery.py \
  --dataset_root /path/to/kaust_mat_dataset \
  --output_json kaust_sparse_recovery_results.json
```

## Аргументы командной строки

| Аргумент | Описание | По умолчанию |
|---|---|---|
| `--dataset_root` | Путь к папке с `.mat`-файлами | обязательно |
| `--output_json` | Путь к JSON-файлу с результатами | `kaust_sparse_recovery_results.json` |
| `--bands` | Целевое число спектральных каналов | `31` |
| `--train_ratio` | Доля кубов для обучения | `0.8` |
| `--sample_pixels_per_train_cube` | Число случайных пикселей из каждого train-куба | `4000` |
| `--sample_pixels_per_test_cube` | Число случайных пикселей из каждого test-куба | `12000` |
| `--n_atoms` | Размер словаря K-SVD | `256` |
| `--sparsity` | Число ненулевых коэффициентов в OMP | `8` |
| `--ksvd_iters` | Число итераций K-SVD | `8` |
| `--response_matrix_npy` | Путь к `.npy`-файлу с матрицей отклика формы `(3, bands)` | `None` |
| `--seed` | Seed для воспроизводимости | `42` |

## Матрица спектрального отклика

Если параметр `--response_matrix_npy` не передан, используется встроенная аппроксимация RGB-чувствительности на основе трёх гладких гауссовых кривых.

Если вы хотите использовать свою матрицу отклика, подготовьте `.npy`-файл формы:

```python
(3, bands)
```

Например для `31` спектрального канала:

```python
(3, 31)
```

## Что делает скрипт

1. Загружает все `.mat`-файлы из `dataset_root`.
2. Для каждого файла:
   - находит гиперспектральный куб;
   - приводит оси к формату `H × W × B`;
   - пересэмплирует спектры до `bands`;
   - нормализует значения в диапазон `[0, 1]`, если максимум больше `1`.
3. Делит кубы на train/test через `train_test_split`.
4. Сэмплирует пиксели из train-кубов.
5. Обучает спектральный словарь `Dh` методом K-SVD.
6. Строит RGB-словарь `D_rgb = R @ Dh`.
7. Для test-кубов:
   - формирует RGB-наблюдения;
   - находит разреженные коэффициенты через OMP;
   - восстанавливает спектры;
   - считает метрики качества.
8. Сохраняет итоговую статистику в JSON.

## Интерпретация метрик

- **RMSE** — среднеквадратичная ошибка восстановления.
- **RRMSE** — относительная RMSE.
- **SAM** (`sam_rad`) — spectral angle mapper в радианах; меньше — лучше.
- **PSNR** — peak signal-to-noise ratio; больше — лучше.

## Ограничения

- Скрипт работает только с `.mat`-файлами.
- Встроенная матрица отклика является приближённой и не заменяет реальную калибровку камеры.
- Пересэмплирование каналов выполняется линейной интерполяцией по нормализованной спектральной оси.
- При большом размере датасета обучение K-SVD и OMP может быть вычислительно затратным.
